Features

Everything you need to understand your data

Codex brings all your Adobe Analytics configuration into one place — with the context and cross-referencing to actually make sense of it.

Complete Configuration Browser

Browse every eVar, Prop, Success Event, List Variable, Segment, Calculated Metric, Processing Rule, and Marketing Channel — all in a fast, searchable, filterable interface with CSV export.

  • eVars & Props
  • Success Events
  • List Variables
  • Segments
  • Calculated Metrics
  • Processing Rules

Data Lineage Tracing

Every dimension detail page shows exactly which Processing Rules, Marketing Channel Rules, and Adobe Launch rules reference it — with match type badges and variable assignment details.

30-Day Trend Charts

Dimension-scoped trend data for every eVar, Prop, and Event — scoped precisely to dimension-specific hits, not total traffic. Powered by the Adobe Analytics API 2.0.

Background Pre-Caching

All configuration data is pre-warmed at startup and refreshed every 24 hours in the background. No waiting — every page loads instantly from cache with a dedicated cache management interface.

API Explorer

Interactive debug pages for all Adobe Analytics API 1.4 and 2.0 endpoints, plus the Adobe Launch Reactor API — browse, inspect parameters, and fire read-only proxied requests from the browser.

Dimension Notes & Tags

Annotate any dimension with plain-English descriptions, technical context, and searchable tags — stored locally and available across your whole team without requiring an external CMS.

Data Lineage

Trace every variable to its source

When you need to know where a variable gets its value, Codex gives you the full picture — not just what's configured, but how it got there.

  • Adobe Launch / Tags See which rules set the variable, with exact variable→value assignments and component type badges (Action, Condition, Event).
  • Processing Rules Which rule sets read from or write to the dimension — with conditions and actions formatted for readability.
  • Marketing Channel Rules Which channel rules reference the dimension, showing attribution logic at a glance.
  • Segments & Calculated Metrics Every segment and metric that references this variable — including transitive references through nested containers.

Get Started

Up and running in minutes

Codex runs locally with a simple Python setup. Connect your Adobe Analytics credentials and you're ready to go.

Clone the repository and install dependencies

Codex uses uv for fast, reliable Python dependency management. Install uv first if you haven't already.

Terminal
$ git clone https://github.com/maxisdigital/codex.git
$ cd codex
$ uv sync
# All dependencies installed — no virtualenv setup needed

Configure your Adobe Analytics credentials

Copy the configuration template and fill in your Adobe Analytics API credentials from the Adobe Developer Console.

Terminal
$ cp config.dist.json config.json
$ nano config.json
config.json
{
  "APP_TITLE":          "My Company",
  "AW_REPORTSUITE_ID": "mycompany.prod",
  "API_VERSION":       "2.0",
  "CLIENT_ID":         "your-client-id",
  "CLIENT_SECRET":     "your-client-secret",
  "ORGANIZATION_ID":   "ABC123@AdobeOrg"
}

Start Codex and verify your setup

Run the health check first, then start the Flask server. Codex begins pre-caching your configuration data on startup.

Terminal
# Verify your setup first
$ uv run verify_setup.py
✓ Config loaded · ✓ API reachable · ✓ All checks passed
$ uv run run.py
Running on http://127.0.0.1:5010
Prefer Docker?
$ docker compose up -d --build
# Available at http://localhost:5010
